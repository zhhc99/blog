比起 "降压" (Undervolting, UV), 人们更常说 "超频" (Overclocking, OC). 它们能帮你给硬件降温, 并释放更多算力.

圈子里的人讲来讲去, 要么复杂化, 要么黑箱化. 还是让我来简单聊聊.

## 降压和超频基本是一回事

不管是显卡还是 CPU, 我们希望它频率 (F) 尽量高. 为了支撑频率, 需要给足能源, 这里也就是电压 (V). 频率越高, 需要的电压就越高.

厂商给你的核心预置一条 V-F 曲线, 一个电压对应一个频率. 每个芯片的品质不同, 所以厂商一般会写个保守的数值, 例如 1.0V 对应 1GHz, 1.1V 对应 1.2GHz.

但, 你的核心很强, 1.0V 就能支撑 1.2GHz. 于是你告诉核心: 把频率提高 0.2GHz 吧. 于是你在 1.2GHz 的负载下节省了 0.1V. **频率的提高叫超频, 电压的节省叫降压, 也叫负压.** 这就是所谓的 "降压超频" 了.

在**负压**时, 如果你的核心只是普普通通, 1.0V 不够支撑 1.2GHz, 你仍然那样指挥核心, 会怎样? 核心不会损坏, 但会算错: 蓝屏, 花屏, 抽帧, 然后自动崩溃了. **负压下硬件是安全的.**

另一种糟糕的指挥是**正压**: 让它在 1.0V 下跑 0.9GHz. 看似是降频, 其实是加压. Intel 13/14 代的缩缸事件就是电压给太高导致的. **加压是真的会让硬件快速老化然后报废的.**

硬件的功耗正比于 V×V×F: 相同频率下降低电压, 对节省功耗很划算.

## 怎么做

根据你的硬件:

| 硬件       | 如何负压                                                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| AMD CPU    | BIOS 里的 Curve Optimizer / [Ryzen Master](https://www.amd.com/en/products/software/ryzen-master.html)                             |
| Intel CPU  | 不同主板不同, 单独查询 / [XTU](https://www.intel.com/content/www/us/en/download/17881/intel-extreme-tuning-utility-intel-xtu.html) |
| NVIDIA GPU | MSI Afterburner ("小飞机") / nvidia-settings                                                                                       |

- CPU 方面, 工具写得很成熟, 用上述 BIOS 或官方软件自带的电压偏移就行.
- 显卡才是超频空间大的那个, 但工具很不成熟, 依赖第三方软件. 小飞机拉半天 V-F 曲线, 无非就是加个频率, 然后把后面拉平在某个频率. **限制最大频率对显卡还挺重要, 主要是为了省电和稳定.**

_要不, 试试 [gpuctl](https://github.com/zhhc99/gpuctl), 几条命令就能搞定显卡的负压了..._

对了, 至于要超多少: **每个硬件都不一样, 强烈建议搜一搜.** 真的是得搜一搜: 找社区的实测内容, 不要问 AI.

- **对于 AMD CPU, 在 BIOS 看自己的 SP 分数. 你只需要知道类似 SP 分数的人在 Curve Optimizer 给多少负压.**
- **对于 GPU, 你只需要知道: 增加的频率, 以及 V-F 曲线拉平时的最大频率.**

当然你也可以自己测: 一点点加频率或负压, 然后去研究怎么用 [OCCT](https://www.ocbase.com/) 做测试. 只跑个 R23 或 Furmark 其实不太专业. 自己摸索数值极限其实是超频的乐趣所在, 可惜这时代下有这闲功夫的人很少了.
