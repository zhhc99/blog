"降压超频" 听起来让人不安. 可事实上, 为 CPU / GPU 降压超频不仅提高性能, 对硬件也是有益无害. 我想用 5 分钟来解释:

- **What**: 什么是降压超频
- **Why**: 降压超频的好处
- **How**: 如何降压超频

## 降压超频的概念和好处

以 CPU 为例 (GPU 本质也是一种 CPU). 我们实际上在意三个参数:

- 性能 (越快越好!)
- 功耗 (越低越好!)
- 稳定性 (不能产生计算错误!)

为此我们需要了解一点点 CPU 的**工作原理**: CPU 通过开关晶体管来工作. 晶体管开关的频率越高, CPU 的性能也就越强; 另外, CPU 的电流大致和电压与频率的乘积成正比. 这也就是说:

$$
\begin{align*}
  P &\propto v^2 f \\\\
  S &\propto f
\end{align*}
$$

这里 $v$ 和 $f$ 分别表示电压和频率, $P$ 和 $S$ 分别表示功耗和性能.

此外, 为了保持稳定性, 当频率越高时, 施加的电压也需要随之增高, 否则电力不足以让晶体管及时完成开关, 从而产生计算错误. 为了满足这一点, 制造商为每个频率 $f_i$ 都指定了一个充分高的电压 $v_i$ 以确保计算正确. 将这些散点连接起来, 就构成了一条 **v-f 曲线**.

![4060 的 v-f 曲线](vf-curve.excalidraw.png "某厂商 GeForce RTX 4060 的 v-f 曲线大致如图.")

- **降压**就是为每个 $f_i$ 指定低于出厂值 $v_i$ 的电压, 即将曲线向左移动. 如您所见, 由于公式里 $v$ 为平方项, **降压能显著降低相同性能下的功耗**.
- **超频**则是让每个电压 $v_i$ 处的频率超出原厂值 $f_i$, 即将曲线向上移动. 显然, **超频能提高相同电压下的性能**.

> 实际上, 无论曲线向左还是向上移动, 最终的结果都是**既降压又超频**. 从定义上想想为什么!

这两个操作可行, 有一个重要原因: **厂家的 v-f 曲线往往非常保守**. 有的 CPU 的体质好, 这里的调整空间较大, 人们称其 "大雕", 而反之则为 "大雷".

有时我们希望 CPU 更强一点, 不太在意功耗; 有时我们希望 CPU 功耗降低且不影响最大性能. 这都可以通过降压超频来实现.

## 如何降压超频

CPU 和 GPU 的调教方式不太相同, 但最终归结到对 v-f 曲线上某些点的修改. 多数用户不需要做这么精细, 只需大概偏移整条曲线就有不错的效果. 以下是一些常用的降压超频操作:

| 设备       | 操作                                                 | 效果     |
| ---------- | ---------------------------------------------------- | -------- |
| AMD CPU    | BIOS 设置负值 Curve Optimizer (不同主板可能名称不同) | 降压     |
| AMD CPU    | BIOS 启用 Precision Boost Overdrive                  | 超频为主 |
| Intel CPU  | XTU 软件中一键超频                                   | 降压超频 |
| Intel CPU  | BIOS 设置负值 Voltage Offset (不同主板可能名称不同)  | 降压     |
| Nvidia GPU | Afterburner 等软件中设置正值 Clock Offset            | 超频     |
| Nvidia GPU | Afterburner 等软件中限制最大频率                     | 降压     |
| Nvidia GPU | Nvidia App 软件中一键超频                            | 降压超频 |

对于每个 CPU 或 GPU, 上述操作的具体数值都不固定. 我们可以在网上找个温和的数值直接用, 多数情况都没什么问题. 如果还是感到不放心, 可以用 [OCCT](https://www.ocbase.com/) 测试稳定性. 如果真的未通过, 那才需要降低数值. 当然, 我们也可以通过反复的 OCCT 测试来自行摸索数值极限, 这也是超频的乐趣.

> 各种一键超频往往不保证稳定性, 建议用 OCCT 测试一下.

- 例如, 9000 系 AMD CPU 往往能接受 `CO-20`, 7000 系的 `CO-10` 也是十拿九稳的数字.
- 再比如说, 多数 Nvidia RTX 4060 都能接受 +150MHz 左右的 Clock Offset. 对于我的这块 Gigabyte RTX 4060 Low Profile, 搜素引擎和 GPU-Z 都指出其默认 Boost Clock 为 2475MHz. 即便我不知道具体的 v-f 曲线, 这也意味着当我采用 `(+150MHz, <=2475MHz)` 的超频配置时, 功耗在任何场合下都优于默认配置. 事实上, 我最终调教得到 `(105W, +240MHz, <=2670MHz)` 的配置. 相比于默认的 `(115W, +0MHz, <=2475MHz)`, 这张显卡用 91% 的功耗获得了 107% 的性能 (理论值).
